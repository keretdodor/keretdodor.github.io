Streaming Website

This project involves deploying an API and a Next.js frontend using Docker containers and running them on Kubernetes with Minikube on my local machine. I also set up a Jenkins container with an agent that automatically updates and deploys the infrastructure whenever a push or pull request is made to GitHub. Additionally, I deployed an ArgoCD node to manage and apply application updates automatically.

The pipeline constantly checks the code's integrity and ensures only valid changes are deployed.

For the project, I created three repositories:

  • Frontend Repository: Contains the Next.js frontend, packaged as a Docker image.
  • API Repository: Hosts the backend API, also packaged as a Docker image.
  • Infrastructure Repository: Manages Jenkins pipelines, development and production environments, and all Kubernetes deployment files.

Telegram Bot - PolyBot

This project is hosted on AWS and involves building a Telegram bot integrated with an AI model. The bot receives an image from a user, analyzes it, and responds with the number and types of objects detected in the image.

AWS services used:

EC2, Auto Scaling, IAM, AMI, DynamoDB, S3, SQS, Secrets Manager, CloudWatch, Route 53.

The entire infrastructure is managed using Terraform and CI/CD pipelines via GitHub Actions, enabling the deployment from scratch.

The project is divided into three repositories:

  • Infrastructure Repository: Manages AWS resources and infrastructure.
  • Bot Repository: Builds and deploys the Telegram bot's Docker image.
  • YOLO Repository: Builds a Docker image for the AI model, converts it to an AMI, and deploys it using Auto Scaling and launch templates.

Election Website with Cloud SQL

This project utilizes Google Cloud Platform (GCP), specifically Google Kubernetes Engine (GKE) and Cloud SQL, to build a voting website. The website, built on WordPress, allows users to vote for a U.S. presidential candidate. The vote data is stored in a Cloud SQL database and displayed on the site with real-time updates.

The infrastructure is deployed using Terraform. The entire environment is private, with connectivity managed through an NGINX Ingress.
