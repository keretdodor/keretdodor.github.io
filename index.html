GCP MongoDB & GKE

This project deploys a highly available and fault-tolerant architecture on GCP using Terraform. It provisions a secure environment featuring a MongoDB replica set and a GKE cluster, each hosted on private subnets within a shared VPC. Emphasizing infrastructure as code, least privilege principles, and best practices for scalability, and network security.

  • Secure Architecture: Private subnets for MongoDB and GKE, with public access via Bastion Host.
  • Scalability: Horizontal Pod Autoscaler and spot node pools ensure cost-effective scaling.
  • Automation: Modularized Terraform code for efficient and reusable infrastructure provisioning.
  • Enhanced Access: TLS-based ingress and Identity-Aware Proxy (IAP) for secure application and Argo CD integration.

Polybot: AI-Powered Telegram Bot

The Polybot Project is a Telegram bot built with Python and integrated with a REST API utilizing the YOLOv5 AI model for image analysis. It is deployed on AWS with Terraform, ensuring high availability and fault tolerance across multiple Availability Zones. The system features CI/CD pipelines for streamlined deployment and efficient development workflows.

  • Deployed on AWS using Terraform modules for scalability and fault tolerance across multiple Availability Zones.
  • Telegram bot integrates with AWS services like S3, SQS, and DynamoDB to handle image uploads, analysis, and results.
  • Dockerized infrastructure with Polybot and YOLOv5 instances, utilizing container management and version control through Docker Hub.
  • CI/CD pipelines automate infrastructure provisioning and application deployment, ensuring consistency and streamlined operations.
